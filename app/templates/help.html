🚀 Центр разработки

Создавайте микросервисы правильно. Соблюдайте различия между путями к файлам (Backend) и ссылками (Frontend).

📋 4 Железных правила
  • 1. Файлы и БД (Python) → Абсолютные

    Systemd может менять рабочую папку.
    BASE_DIR = os.path.dirname(os.path.abspath(__file__))
    db_path = os.path.join(BASE_DIR, 'data.db')

  • 2. Ссылки и Fetch (JS/HTML) → Относительные

    Приложение работает за прокси (/proxy/appname/).
    ❌ Нельзя: fetch('/api/data') (идет в корень)
    ✅ Надо: fetch('api/data') (идет в папку приложения)

  • 3. Хост и Порт

    Хост строго 127.0.0.1.
    Порт из ENV: os.environ.get('PORT').

  • 4. Запрет Reload

    app.run(..., debug=False, use_reloader=False)

🤖 Исправленный Генератор (Промпт)

Этот промпт теперь учит нейросеть писать правильные fetch запросы без слеша в начале.

🛠 Пример: как работают пути
# Backend (Python) - Абсолютные пути к файлам
BASE_DIR = os.path.dirname(os.path.abspath(__file__))
conn = sqlite3.connect(os.path.join(BASE_DIR, 'base.db'))

# Frontend (HTML/JS) - Относительные ссылки
# Браузер сам подставит /proxy/appname/ перед api/users
html = """
<script>
  // Обратите внимание: нет слэша перед 'api'
  fetch('api/users')
    .then(r => r.json())
    .then(data => console.log(data));
</script>
"""
